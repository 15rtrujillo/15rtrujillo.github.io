OpenRSC

OpenRSC is an open-source project that aims to reproduce and preserve RuneScape Classic, a massively multiplayer online role-playing game developed by Jagex. The game was shut down in August 2018.

In the months leading up to its decomission, several players set out to preserve as much information about the game as possible. This was mostly done by using an open-source, third-party client known as RSC+, which had the ability to capture the packet data being sent from and recieved by the client.

Using these packet captures, the decompiled Java client, and any other information that was recorded about the game, OpenRSC boasts a 98% accurate reporduction of the original game. While this "98%" number might not be wholly accurate, OpenRSC is the most accurate representation of the game as it was before its shutdown in 2018.

I joined the OpenRSC project in early 2020, before the COVID lockdowns. I joined mostly because I was interested in learning about online games and how they worked. At the time, I was only in my second year of my computer science degree and didn't really know how client-server multiplayer games worked. This project has taught me a lot, not only about client-server applications and games, but also about working on projects with a team of people.

Over the years I have contributed a lot to OpenRSC (as you can see on my Github page). Mostly I've done bugfixes, authenticity corrections, and content for the custom "Cabbage" server. I have also done a few larger, engine-focused projects as well.

OpenRSC mostly consists of three "modules." There's the server (the game engine), "plugins" (the game content, like quests and object/NPC interactions), and the client (which can be built for PC or Android). All of these modules are written in Java, and I have contributed to all of them multiple times during my time with the project.
